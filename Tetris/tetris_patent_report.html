一种基于分阶段模仿学习与共享棋盘强化学习的俄罗斯方块对抗智能体训练方法

（一）技术领域

本发明涉及强化学习、模仿学习、博弈型游戏智能体训练以及共享棋盘交互系统技术领域，具体涉及一种面向俄罗斯方块共享棋盘对抗场景的分阶段训练方法。该方法通过启发式教师蒸馏、玩家落点微调以及共享棋盘强化学习三阶段递进训练，得到能够在真实对抗规则下稳定运行的游戏智能体。

（二）背景技术

俄罗斯方块是典型的离散决策问题，其智能求解通常有两类主流方法：一类是手工构造评估函数的启发式搜索方法；另一类是利用深度学习或强化学习进行策略近似。前者具有可解释性强、实现简单等优点，但策略上限受到人工特征设计限制，且难以自然适配新规则；后者具备继续提升和自动优化的潜力，但在训练初期存在奖励稀疏、动作空间大、收敛不稳等问题。

对于本项目所采用的共享棋盘对抗规则，仅使用传统启发式搜索存在两个突出不足：第一，启发式模型可在静态棋盘上选择较优落点，但难以学习玩家风格，也难以将已有经验迁移到新的对抗性奖励目标中；第二，若直接从零开始做强化学习，智能体需要同时学习基本堆叠策略、共享棋盘规则、碰撞交互以及得分导向行为，训练开销大且极易出现“长时间不消行、不会赢”的停滞现象。

进一步地，若将强化学习环境简化为单机俄罗斯方块，那么训练目标与实际部署环境之间会出现不一致：训练中 AI 面对的是单活动方块、单棋盘、单方得分，而部署时却是共享棋盘、双活动方块、局面对手实时变化、碰撞权状态可切换的博弈环境。该差异会造成模型在离线训练中表现良好，而在真实对战中难以稳定取胜。因此，需要一种兼顾“先学会下、再学玩家、最后学会赢”的分阶段训练方法，使智能体既能继承基础策略，又能逐步适应共享棋盘下的真实胜负目标。

（三）发明内容

1. 发明目的

本发明的目的在于提出一种基于分阶段模仿学习与共享棋盘强化学习的俄罗斯方块对抗智能体训练方法，以解决现有技术中启发式模型难以突破上限、玩家模仿模型难以超越示范者、以及直接强化学习难以在共享棋盘环境中稳定收敛的问题，最终获得相对于启发式 AI 与玩家操作都更具竞争力的对抗智能体。

2. 技术方案

为实现上述目的，本发明采用如下技术方案：首先构建共享棋盘游戏系统，并将智能体动作定义为“当前方块的最终落点”，而非离散按键序列；其次引入三阶段训练链路，即利用启发式教师生成大规模落点标签预训练模型一、利用玩家实际对局数据对模型一进行风格微调得到模型二、再在共享棋盘真实规则下以模型二为初始化开展强化学习得到模型三；最后将模型三部署到统一菜单入口的人机模式中，并在共享棋盘局面变化时实时触发落点重规划。

在上述训练链路中，模型一、模型二和模型三均采用同一种 afterstate 价值网络作为基础网络结构。该网络不是直接输入当前局面并输出固定编号动作，而是对每一个“候选最终落点执行后的棋盘状态”进行打分；系统先枚举当前方块所有合法旋转和横坐标，再将每个候选落点转换为特征向量，最后由网络输出单个价值标量并选取分数最高的落点。由此，监督学习阶段、玩家模仿阶段和强化学习阶段可以共用同一套状态表示和模型参数，避免不同阶段之间因动作编码不同而产生额外转换误差。

模型三的强化学习训练进一步采用在线价值网络与目标价值网络配合的 DQN 式更新方式，并结合经验回放缓冲区、epsilon-greedy 探索策略和周期性目标网络同步。在线网络用于选择并更新当前落点价值，目标网络用于计算较稳定的时序差分目标，从而降低共享棋盘对抗环境中比分、碰撞、消行等反馈剧烈变化造成的训练振荡。

步骤六：将最优模型部署到实际人机模式中，并在共享棋盘发生局面变化时执行实时重规划。

3. 创新点

创新点一：提出一种“启发式蒸馏—玩家微调—共享棋盘强化学习”的三级递进式训练框架。 该框架不是直接让强化学习从随机策略开始搜索，而是先利用启发式教师生成大规模稳定数据，使模型掌握基本堆叠能力；再引入玩家最终落点数据，使模型获得符合真实操作习惯的风格偏置；最后在共享棋盘环境下通过强化学习进一步提升胜率和对抗能力。该方法显著降低了共享棋盘 RL 的冷启动难度，避免了训练初期长期无效探索的问题。

创新点二：提出一种面向共享棋盘的宏动作 afterstate 强化学习训练方法。 本发明并非采用“左、右、旋转、下落”等按键级动作进行强化学习，而是将一个强化学习动作定义为“选择当前方块的最终落点方案”，环境内部再自动执行至该方块落锁。与此同时，奖励函数不仅考虑得分与胜负，还结合分差、消行、洞数、高度和凹凸度等棋盘质量指标，并在局面对手变化时触发重新规划，从而更贴近共享棋盘对抗规则下的真实决策逻辑。

创新点三：在同一 afterstate 价值网络上统一实现教师蒸馏、玩家模仿和强化学习微调。 传统游戏智能体常将模仿学习网络、强化学习网络和部署策略分别设计，导致训练阶段之间难以平滑衔接。本发明采用多层感知机价值网络对候选落点进行统一评分，使启发式标签、玩家标签和强化学习回报都能作用于同一个标量价值函数。该设计既保留了启发式特征的可解释性，又使网络具备通过数据继续优化的能力。

4. 有益效果

与现有技术相比，本发明至少具有如下有益效果：第一，通过将启发式策略蒸馏为神经网络模型，可在保留基础策略质量的前提下为后续强化学习提供稳定初始化；第二，通过玩家数据微调，可使模型在不完全丧失基础能力的情况下学习真实人类的落点偏好；第三，通过在共享棋盘环境中进行强化学习，模型训练目标与实际部署目标保持一致，避免了“训练环境强、实战环境弱”的迁移失配；第四，通过宏动作 afterstate 设计显著压缩了动作空间，提高了训练效率；第五，通过阶段化训练与最优 checkpoint 选择，使最终部署模型在对抗成绩上优于中间训练阶段模型。

进一步地，所述 afterstate 价值网络具有输入维度明确、推理速度快、候选动作可解释等优点。与卷积网络直接从图像或棋盘矩阵端到端学习相比，本方法将棋盘二值占用、列高度、洞数、总高度、凹凸度和下一方块编码组合为结构化特征，使网络更容易捕捉俄罗斯方块中影响局面质量的关键因素；与按键级策略网络相比，本方法一次决策即可确定最终落点，减少了长动作序列带来的误差累积，特别适合需要实时响应玩家干扰的共享棋盘场景。

（五）具体实施方式

实施例一：训练总体框架

本实施例中，训练系统由教师数据生成模块、玩家数据采集模块、监督学习训练模块、共享棋盘强化学习模块、模型评估模块和最终部署模块组成。教师数据生成模块通过启发式搜索自动生成最终落点标签；玩家数据采集模块记录玩家真实对局中的最终落点决策；监督学习训练模块用于得到模型一和模型二；强化学习模块在共享棋盘环境中继续提升模型；评估模块负责记录不同 checkpoint 的胜率、得分和分差；部署模块负责将最优模型接入正式游戏入口。

在代码实现中，启发式教师由 TetrisAI 负责，根据总高度、洞数和凹凸度构造评估函数生成稳定落点；神经网络由 AfterstateValueNet 负责，作为模型一、模型二和模型三的统一价值函数；RLTetrisAI 在部署阶段加载 checkpoint，对所有合法落点批量构造 afterstate 特征并调用价值网络评分；SharedBoardTetris 则提供共享棋盘、双活动方块、碰撞权、消行得分和胜负判定等环境接口。因此，训练和运行并不是孤立模块，而是围绕同一个落点价值网络和同一套共享棋盘状态持续递进。

从项目技术栈看，本实施例主要采用 Python 作为开发语言，使用 Pygame 完成窗口创建、图形绘制、键盘输入、菜单切换和游戏循环控制，使用 PyTorch 构建 afterstate 价值网络、加载与保存 .pt checkpoint，并在强化学习训练中完成前向推理、损失计算和参数更新。项目未使用 Java、JavaScript、Vue、React、Express 或 Three.js 等 Web 前端/后端框架，整体属于桌面端 Python 游戏与深度学习训练程序。

开发与运行工具方面，项目使用 Conda/Python 虚拟环境管理依赖，通过 VS Code 等编辑器进行代码编写和调试，通过命令行脚本完成模型训练、断点续训和人机模式启动；Python 的 py_compile 可用于基础语法检查，LibreOffice/WPS 可用于课程报告的查看和导出。数据存储方面，项目没有引入 MySQL、SQLite、MongoDB 等数据库，训练模型以 checkpoint 文件形式保存在本地目录，玩家数据和训练命令以本地文件记录，背景图片等静态资源也以普通文件方式存放。这种轻量级文件存储方式适合课程项目快速迭代，减少了部署数据库服务的额外成本。

实施例二：模型结构与状态表示

本实施例中，智能体核心网络采用 afterstate 价值网络，即输入一个候选最终落点对应的棋盘特征向量，输出该落点的价值分数。所述特征向量包括棋盘二值展开、各列高度、洞数、总高度、凹凸度以及下一方块 one-hot 编码等信息。网络主体采用多层感知机结构，隐藏层宽度设置为256和256，输出为单个标量价值。部署时，系统对当前方块的所有合法最终落点进行枚举，由价值网络选取最优方案，再逐步生成旋转、横移和到底动作。

具体而言，在本项目 25×25 的共享棋盘设置下，棋盘二值展开得到625维特征，25列高度得到25维特征，洞数、总高度和凹凸度得到3维统计特征，下一方块 one-hot 编码得到7维特征，合计输入维度为660。网络结构为 Linear(660,256)-ReLU-Linear(256,256)-ReLU-Linear(256,1)，可训练参数约为23.5万个。该规模相对于大型深度网络更轻量，能够在每次决策时同时评估多个候选落点；同时又比纯手工打分函数更灵活，可以通过教师数据、玩家数据和强化学习回报持续调整权重。

所述网络没有采用卷积神经网络或循环神经网络，原因在于本任务的关键决策不是连续图像识别或长时间序列预测，而是对有限数量的最终落点进行局面质量比较。多层感知机配合 afterstate 特征可以直接吸收列高度、洞数、凹凸度等强先验，使训练样本利用率更高，并降低共享棋盘实时推理时的计算开销。

实施例三：模型一训练

本实施例中，首先利用启发式搜索 AI 自动对局，构建大规模教师数据集。每条样本记录当前棋盘状态、当前方块形状、下一方块信息以及教师 AI 最终选择的落点。训练时，以教师实际选择的落点为正样本，在候选落点集合上进行监督学习，使模型一学会启发式搜索策略。模型一的作用不在于直接超过启发式 AI，而在于为后续微调和强化学习提供具备基本堆叠能力的初始化参数。

在模型一训练中，启发式 AI 相当于“教师网络之外的规则教师”，其优势是标签生成稳定、成本低，并能覆盖大量基础局面。对每个训练样本，系统枚举候选落点并为每个候选落点生成660维 afterstate 特征，模型通过提高教师落点的相对价值、降低非教师落点的相对价值来完成行为克隆。由于模型一学习的是“落锁后的局面价值”而非单步按键，因此不会受到玩家按键节奏、输入延迟或中途修正动作的干扰。

实施例四：模型二训练

本实施例中，在模型一基础上引入玩家真实对局数据进行微调。与按键级模仿不同，本实施例采集的是玩家“最终把方块放在哪里”的落点标签，从而降低单帧按键监督中的噪声。通过该步骤，模型二在保留基础堆叠能力的同时，能够学习玩家在共享棋盘环境下的策略偏好。实际训练中，模型二相对于仅用少量玩家数据从零训练的方法表现更稳定，证明模型一提供的先验对于后续学习具有明显帮助。

模型二继续沿用模型一的 afterstate 价值网络参数，并通过玩家数据进行二次监督训练。项目训练记录中，该阶段以模型一 checkpoint 作为初始化，采用玩家 actor 数据、30个训练轮次、批量大小8和学习率1e-4进行微调。其优点在于：一方面，模型不会因为玩家样本数量有限而完全遗忘启发式教师提供的基础堆叠能力；另一方面，网络权重会向真实玩家在共享棋盘压力下更常选择的落点移动，使后续强化学习的起点更接近实际部署分布。

实施例五：共享棋盘强化学习环境

本实施例中，强化学习环境不再使用单机俄罗斯方块规则，而直接采用共享棋盘、双活动方块、碰撞权和比分判定的真实部署规则。每次强化学习动作不再对应一次按键，而是对应一个最终落点选择；环境在内部自动执行该落点对应的旋转、横移和到底，并在当前方块落锁后返回奖励。奖励由以下部分构成：本步得分增量、与对手的分差变化、消行数量、棋盘洞数变化、总高度变化、凹凸度变化以及胜负终局奖励。这样既保留了共享棋盘对抗的主目标，也利用棋盘质量 shaping 缓解奖励稀疏问题。

在强化学习过程中，模型三不再只模仿某个固定标签，而是把 afterstate 价值网络作为 Q 值近似器使用。在线网络根据当前候选落点输出价值并执行 epsilon-greedy 探索，经验回放缓冲区保存状态、落点、奖励和下一状态，训练时从缓冲区抽取小批量样本；目标网络根据下一状态候选落点的最大价值构造时序差分目标，并按照固定步数从在线网络同步参数。该使用方式能够减少连续对局样本之间的相关性，并让模型在碰撞、消行、对手干扰等非平稳反馈下保持较稳定的更新方向。

实施例六：阶段化强化学习训练

本实施例中，模型三的强化学习分为两个阶段。阶段一采用启发式 AI 作为对手，使模型先适应共享棋盘中的碰撞、得分与终局规则；阶段二采用混合对手，包括启发式 AI、玩家微调后的模型二以及历史最优模型快照，使训练目标从“适应环境”进一步过渡到“对抗更强对手”。训练过程中，以滚动窗口平均分差、平均得分和平均获胜局数作为 checkpoint 选择指标，而非仅依赖单局奖励值。

阶段二训练记录表明，模型三从阶段一最优 checkpoint 初始化，并继续使用隐藏层为256、256的 afterstate 价值网络；强化学习参数包括折扣因子0.99、学习率5e-5、经验回放容量20000、预热步数256、目标网络同步间隔200、训练间隔16、探索率由0.2逐步衰减至0.03、单局最大步数120。上述设置的目的在于让网络在保持已有落点能力的基础上进行较小幅度的对抗微调，避免学习率过大或探索过强造成已学会的堆叠策略被破坏。

实施例七：实验结果

在本实施例中，阶段一训练结束后，模型三已经能够在共享棋盘环境中稳定得分，但平均分差仍未转正；在阶段二训练后，模型三在150局对抗中取得智能体总分48分、对手总分44分、平均分差+0.0267、获胜21局的结果，说明其已在整体表现上超过阶段一版本，并具备替换原始模型二进入正式部署的条件。最终部署时，系统采用阶段二训练中保存的最优 checkpoint 作为正式人机模式的默认模型。

表1　本发明实施例的训练阶段与效果概述

阶段 训练方式 主要作用 结果特征
模型一 启发式教师数据监督学习；使用 afterstate MLP 价值网络学习教师落点 学习基本堆叠与落点选择能力 为后续微调和强化学习提供稳定初始化
模型二 玩家最终落点数据微调；在模型一网络参数上继续行为克隆 学习玩家偏好与共享棋盘操作风格 在实战中表现稳定，成为强化学习初始化基础

模型三阶段二 共享棋盘 RL，对手为混合对手；在阶段一 checkpoint 上继续微调 提高对抗能力并逼近最终部署目标 150局中总分48:44，平均分差+0.0267，获胜21局

综上，本发明将俄罗斯方块共享棋盘对抗问题明确转化为“先学会下、再学玩家、最后学会赢”的分阶段训练问题，并通过 afterstate 宏动作设计、共享棋盘强化学习环境与阶段化对手策略，获得了能够实际部署的人机对抗模型。该方法不仅适用于本项目，也可为其他规则复杂、奖励稀疏、部署环境与训练环境差异较大的游戏智能体训练任务提供参考。
